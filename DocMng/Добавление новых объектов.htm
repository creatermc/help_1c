Добавление новых объектов в конфигурации, содержащие БИД

При добавлении новых объектов метаданных в конфигурации, содержащие Библиотеку интеграции с 1С:Документооборотом, следует:

1.             Определить, понадобится ли пользователям возможность интеграции с 1С:Документооборотом по этим объектам, и в каком объеме для этих объектов могут быть востребованы возможности 1С:Документооборота.

2.             При необходимости включить эти объекты в тип параметра общих команд Библиотеки, внести изменения в модули форм этих объектов и в переопределяемые общие модули.

1. Нужна ли интеграция?

Библиотека поддерживает:

1.              Хранение присоединенных файлов в 1С:Документообороте.

2.              Создание в 1С:Документообороте архива документов, созданных в интегрируемой конфигурации.

3.              Запуск бизнес-процессов, например, согласования, на основании объектов интегрированной системы.

4.              Создание исходящих писем на основании этих объектов.

5.              Учет трудозатрат и хронометраж.

Для большинства прикладных справочников и документов окажется востребованной хотя бы одна из этих возможностей. Как правило, в интеграции не участвуют лишь скрытые от пользователей объекты и те объекты, с которыми взаимодействуют только администраторы.

2. как это выглядит в пользовательском интерфейсе?

Типичная форма интегрированного объекта (на примере Заказа клиента из 1С:ERP Управление предприятием):

В панели навигации есть команда Документооборот, в подменю создания на основании – команды создания процессов и писем, в командной панели – кнопки хронометража. Кроме того, подменю печати может содержать команду печати листа согласования, а командная панель – кнопку перехода к списку присоединенных файлов.

3. подключение новых объектов

Большинство действий по подключению новых объектов не зависит от того, какой объем возможностей интеграции представляется потенциально востребованным для объектов этого типа. Исключением является запуск бизнес-процессов, создание писем, быстрый переход к присоединенным файлам и печать листа согласования, которые требуют особого подхода и будут описаны отдельно.

Действия, выполняемые при подключении новых объектов, даны ниже в таком формате:

Тип изменяемого объекта

Изменяемый объект

№

Выполняемые действия

Видимый эффект

Важно: При проверке эффекта от внесенных изменений убедитесь в том, что включены все функциональные опции в панели настройки интеграции.

3.2 Действия при использовании отдельных возможностей интеграции

Следующие действия необходимы для использования бизнес процессов 1С:Документооборота. Для некоторых объектов, которые заведомо будут участвовать исключительно в процессе согласования, можно использовать только специализированную команду (подробнее об особенности запуска процесса согласования см. в документации БИД). Для остальных объектов следует использовать общую команду создания бизнес-процесса:

 

Общая команда

ИнтеграцияС1СДокументооборотСоздатьПроцессСогласования

10

Объект включается в тип параметра команды.

В подменю Создать на основании появляется команда Процесс согласования.

 

Следующие действия необходимы для объектов, по которым просматривается регулярное использование согласования с помощью 1С:Документооборота.

Общий модуль

ИнтеграцияС1СДокументооборотБазоваяФункциональностьПереопределяемый

11

В процедуру ПередДобавлениемКомандПечати по образцу в комментарии добавляется код, для формы добавляемого объекта метаданных, добавляющий команду печати листа согласования.

В подменю печати объекта появляется печатная форма Лист согласования.

 

Общий модуль

ИнтеграцияС1СДокументооборотБазоваяФункциональностьПереопределяемый

12

В процедуру ПриИзмененииСостоянияСогласования добавляется код, автоматически меняющий статус документа интегрируемой конфигурации при изменении состояния согласования в 1С:Документообороте.

Видимых изменений нет. Это избавляет от необходимости настройки изменения статуса на внедрении, и позволяет включить изменение состояния в 1С:Документообороте в бизнес-логику интегрируемой конфигурации, если, например, изменение статуса документа влечет за собой более сложные операции.

 

Следующее действие необходимо для быстрого доступа к присоединенным файлам из формы объекта. Без этого присоединенные файлы будут доступны только из формы Документооборот, открывающейся по команде в панели навигации. Имеет смысл для тех объектов, операции с файлами, которых – частотные и должны быть ближе по кликам.

Общая команда

ИнтеграцияС1СДокументооборотПрисоединенныеФайлы

13

Объект включается в тип параметра команды.

В панели навигации формы объекта появляется команда Присоединенные файлы
